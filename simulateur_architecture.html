Pourquoi une telle différence de vitesse ?

Le Réseau Wi-Fi (Lent)

Le Wi-Fi est un protocole complexe et lourd. Quand le Serveur envoie un message au Tag :

  • Le paquet passe par la couche logicielle de l'ordinateur (Windows).
  • Il voyage jusqu'au routeur (la box internet ou le point d'accès).
  • Le routeur gère les collisions, met le paquet dans une file d'attente.
  • L'ESP32 le reçoit et la couche réseau TCP l'analyse (Handshake, acquittements).

Bilan : Plusieurs millisecondes (voire dizaines de ms) perdues en routage logiciel.

Le Réseau UWB (Instantané)

L'UWB (Puce Decawave) est une technologie de niveau matériel pur (Point-à-Point) :

  • Il n'y a pas de routeur ou de box au milieu. L'antenne tire tout droit vers l'autre.
  • Le message est une simple impulsion électrique ultra-courte.
  • C'est la puce matérielle elle-même qui gère la réception, pas un logiciel !
  • Les horloges internes traitent l'onde à la vitesse de la lumière.

Bilan : L'aller-retour complet prend moins d'1 milliseconde, sans aucune surcouche logicielle.

💡 Conclusion : Le Wi-Fi est génial pour envoyer des données longues distances au Serveur (PC), mais pour le "Temps Réel" local, il faut absolument minimiser son utilisation et laisser les puces UWB travailler toutes seules !
